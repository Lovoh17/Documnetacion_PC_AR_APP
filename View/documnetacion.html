Introducción

Azamoz Shopping es una aplicación de comercio electrónico con realidad aumentada desarrollada en Android Studio con Java.

Requisitos

Desarrollo
  • Android Studio Flamingo | 2022.2.1+
  • JDK 11+
  • SDK Platform Android 8.0+
  • Dispositivo compatible con ARCore
Producción
  • Android 8.0+ (Oreo)
  • 2GB RAM mínimo

  • Conexión a Internet

Arquitectura

Activities

AdminActivity

Activity que proporciona la interfaz de administración del sistema. Permite al administrador gestionar usuarios, productos y órdenes.

  • Funcionalidades: Dashboard administrativo, gestión de usuarios, gestión de productos, visualización de estadísticas
  • Layout: activity_admin.xml
  • Navegación: Accesible solo para usuarios con rol de administrador

AR_test

Activity de prueba para funcionalidades de Realidad Aumentada (AR). Implementa pruebas de anclajes y visualización de objetos 3D.

  • Tecnologías: ARCore, Sceneform
  • Dependencias: com.google.ar.sceneform.ux
  • Notas: Activity de desarrollo/testeo, no para producción

GiftCardStoreActivity

Activity que muestra el catálogo de tarjetas de regalo disponibles para compra.

  • Funcionalidades: Listado de gift cards, filtrado por categorías, compra de tarjetas
  • Integraciones: API de pagos, Firebase Database
  • Eventos: onClick para selección de tarjeta

InicioActivity

Pantalla principal de la aplicación después del login. Muestra contenido destacado y opciones principales.

  • Componentes: RecyclerView para productos destacados, menú inferior
  • Navegación: Conduce a ProductShowARActivity, GiftCardStoreActivity, OrdersActivity
  • Datos: Obtiene información de Firebase Realtime Database

LoginActivity

Pantalla de inicio de sesión para usuarios registrados. Valida credenciales contra Firebase Authentication.

  • Validaciones: Formato email, longitud contraseña
  • Flujo: Redirige a InicioActivity tras login exitoso
  • Opciones: Enlace a RegisterActivity, recuperación de contraseña

OrdersActivity

Muestra el historial de pedidos del usuario con capacidad de filtrar y ver detalles.

  • Vistas: RecyclerView con adaptador personalizado
  • Estados: Muestra pedidos pendientes, completados y cancelados
  • Acciones: Cancelar pedidos pendientes, reordenar

ProductShowARActivity

Activity principal de Realidad Aumentada que permite visualizar productos en 3D en el entorno del usuario.

  • Features AR: Anclajes persistentes (wrappedAnchor), escalado de modelos
  • Modelos 3D: Carga desde Firebase Storage
  • Interacción: Rotación con gestos, modo examen detallado

RegisterActivity

Pantalla de registro para nuevos usuarios. Recopila información básica y crea cuenta en Firebase Auth.

  • Campos: Nombre, email, contraseña, confirmación contraseña
  • Validaciones: Unicidad de email, fortaleza contraseña
  • Flujo: Redirige a LoginActivity tras registro exitoso

SplashActivity

Pantalla inicial de la aplicación con logo y versión. Realiza comprobaciones iniciales.

  • Duración: 2 segundos (mínimo)
  • Lógica: Verifica sesión activa, redirige a LoginActivity o InicioActivity
  • Assets: Logo corporativo, animación de carga

wrappedAnchor

Componente de ARCore que gestiona anclajes persistentes para objetos de realidad aumentada.

  • Propósito: Mantener posición de objetos AR entre sesiones
  • Técnica: Almacena coordenadas relativas y datos de orientación
  • Uso: Implementado en ProductShowARActivity

ARCore con Android Studio (Java)

Implementación de realidad aumentada en la aplicación

1. Configuración inicial

Agrega estas dependencias en tu build.gradle:

dependencies {
    implementation 'com.google.ar:core:1.35.0'
    implementation 'com.google.ar.sceneform:core:1.17.1'
    implementation 'com.google.ar.sceneform.ux:sceneform-ux:1.17.1'
}

2. Configura el AndroidManifest.xml

<uses-permission android:name="android.permission.CAMERA" />
<uses-feature android:name="android.hardware.camera.ar" android:required="true" />

<application>
    <meta-data android:name="com.google.ar.core" android:value="required" />
</application>

3. Implementación básica

Crea una actividad AR:

public class ARActivity extends AppCompatActivity {
    private ArFragment arFragment;
    
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_ar);
        
        arFragment = (ArFragment) getSupportFragmentManager()
            .findFragmentById(R.id.arFragment);
            
        // Configuración de modelos y listeners...
    }
}
Consejo: Usa Sceneform para manejar modelos 3D de manera eficiente.
